\subsubsection{GXL Import und Export Funktionalität}
Da bei der Erstellung dieses Dokumentes noch kein Übereinkommen mit 
dem Auftraggeber getroffen werden konnte, und dieses auch erst in den nächsten Wochen möglich sein wird,
wird in diesem Fall nur eine sehr grobe Spezifikation vorgenommen. 
Grundsätzlich ist vorgesehen, dass der GXL Import / Export in einem separaten Plug-In realisiert werden wird, das zum Feature-Set gehört. 
Es ist vorgesehen, sowohl die Daten des Architekturgraphen, als auch die 
Daten, die auf Dateiebene hinter der Darstellung stehen, exportieren zu können. 
So werden bei einem Export die Architektur-Daten in Form einer XML Datei, 
die dem mit dem Kunden noch zu spezifizierenden GXL Schema entspricht, gespeichert.
Die mit der Architektur zusammenhängenden Assets werden in einem .jar File exportiert. Dies schließt
Meta Informationen aus.
