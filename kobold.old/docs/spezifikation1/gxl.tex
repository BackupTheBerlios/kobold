\subsubsection{GXL Import und Export Funktionalität}
Da bei der Erstellung dieses Dokumentes noch keine detaillierte Einigung
mit dem Auftraggeber getroffen werden konnte, wird für die Spezifikation
I nur sehr grob spezifiziert, wie der GXL Import/Export realisiert
werden soll.\par
Der GXL Import/Export wird in einem separaten Plug-In realisiert, das
zum Kobold-Client 'Feature-Set' gehört.\par
Es ist vorgesehen, sowohl die Daten des Architekturgraphen, als auch die 
Daten, die auf Dateiebene der Darstellung zugrundliegen, exportieren zu können. 
\par
So werden bei einem Export die Architektur-Daten in Form einer XML Datei, 
die dem mit dem Kunden noch zu spezifizierenden GXL Schema entspricht, gespeichert.
Die mit der Architektur zusammenhängenden Sourcen werden in einem .jar File exportiert.
Die Kobold Meta Informationen werden dabei ausgeschlossen.
