\section{Architektureditor}

\subsection{Produkt}

\subsubsection{Neues Produkt anlegen}

\usecase{Kobold ist gestartet}
{Der Benutzer öffnet die Palette im Architektureditor und wählt das Werkzeug 'compose product' aus. 
Daraufhin ändern alle Objekte im Editor ihre Farbe in grau. Der Benutzer klick auf die Objekte, die er
in seinem Produkt enthalten haben möchte. Diese färben sich daraufhin blau. Ist der Benutzer mit seiner
Auswahl zufrieden, betätigt er den 'create Product' Button im Editor. Ein Dialog öffnet sich, in dem der 
Benutzer die Metainformationen für das neue Produkt eingeben kann.}
{Alle gewählten Module inklusive ihrer benötigten Referenzen sind ausgecheckt und werden dem
Beutzer grafisch angezeigt.}
{Nicht spezifiziert}
{PLE}


\subsubsection{Produkt umbenennen}
\usecase{Es ist ein Produkt ausgewählt}
{Der Benutzer doppelklickt auf das Produkt im RoleTree. Ein Dialog öffnet sich, in dem er den Namen
 ändern kann. Der Benutzer bestätigt mit 'OK'.}
{Produkt ist umbenannt.}
{Nicht spezifiziert}

\subsubsection{Produkt als veraltet markieren}
\usecase{
Benutzer ist mindestens PLE für diese Produktlinie}
{
Der Benutzer doppeltklickt auf das entsprechende Produkt. Es öffnet sich ein Dialog,
in dem er das Produkt auf deprecated setzen kann. Klickt er hier auf OK ist
das Produkt auf deprecated gesetzt und seine Module stehen den PEs nicht
mehr zur Auswahl.}
{
Produkt ist auf deprecated gesetzt, seine Module stehen für neue Produkte nicht mehr zur 
Verfügung.}
{Nicht spezifiziert}
{PLE}
