
\subsection{Produktlinien}

\subsubsection{Variante erstellen}
\usecase{Produktlinienarchitektur geöffnet}
{Über den Palette wird das Werkzeug ''Variant'' ausgewählt
  und durch einen Klick in eine Komponente eine
  Variante mit fester Größe eingefügt. Mittels Klicken\&Ziehen kann
  zusätzlich die Größe der Variante eingestellt werden. Varianten
  können nur in Komponenten erstellt werden.}
{Es wird ein Dialog angezeigt. In diesen werden die Metadaten für die
  Variante eingetragen.}
{Nicht spezifiziert}
{PLE}

\subsubsection{Komponente erstellen}
\usecase{Produktlinienarchitektur geöffnet}
{Über den Palette wird das Werkzeug ''Component'' ausgewählt
  und durch einen Klick in eine Variante oder in die Architektur dort mit fester Größe
  eingefügt. Mittels Klicken\&Ziehen kann zusätzlich die
Größe der Komponente eingestellt werden. Komponenten können nur top-level oder in
Varianten erstellt werden.}
{Es wird ein Dialog angezeigt. In diesen werden die Metadaten für die
  Komponente eingetragen.}
{Nicht spezifiziert}
{PLE}

\subsubsection{Version erstellen}
\usecase{Produktlinienarchitektur geöffnet}
{Über den Palette wird das Werkzeug ''Release'' ausgewählt
  und durch einen Klick in eine Variante dort mit fester Größe
  eingefügt. Mittels Klicken\&Ziehen kann zusätzlich die 
Größe der Version eingestellt werden. Versionen können in Varianten
erstellt werden.}
{Es wird ein Dialog angezeigt. In diesen werden die Metadaten für die
  Version eingetragen. Dazu können für alle Dateien der Variante die
  Revisionsnummern ausgewählt werden und so zu einer Version gebündelt
werden.}
{Nicht spezifiziert}
{PLE}

\subsubsection{Metaknoten erstellen}
\usecase{Produktlinienarchitektur geöffnet}
{Über die Palette wird das Werkzeug ''AND Node'' bzw. ''OR Node'' ausgewählt und durch
  einen Klick in den Editor-Bereich ein Metaknoten eingefügt.}
{Es öffnet sich ein Dialog. In diesem wird der Typ des Knoten
  ausgewählt. Mögliche Typen sind ''AND'', ''OR''.
% mehr?
}
{nicht spezifiziert.}
{PLE}

\subsubsection{Komponente, Variante, Version löschen}
\usecase{Produktlinienarchitektur geöffnet, Element ausgewählt}
{Über das Kontextmenu wird die Aktion ''Löschen''
  ausgewählt. Alternativ kann auch die Taste ''Del'' bzw. ''Entf''
  gedrückt werden.}
{Es wird überprüft, ob das ausgewählte Element bereits in Produkten
  verwendet wird. Ist dies der Fall, wird dem Benutzer ein Dialog
  angezeigt, der ihn über diese Situation aufklärt und ihm die
  Möglichkeit gibt, den Vorgang abzubrechen, das gewählte Element als
  ''deprecated'' zu markieren oder wirklich zu löschen. Dazu wird es
  auch aus allen Produkten entfernt. Wird das Element nicht verwendet,
wird es aus der Architektur entfernt.}
{Element bleibt in der Architektur.}
{PLE}

\subsubsection{Metaknoten löschen}
\usecase{Produktlinienarchitektur geöffnet, Metaknoten ausgewählt}
{Über das Kontextmenu wird die Aktion ''Löschen''
  ausgewählt. Alternativ kann auch die Taste ''Del'' bzw. ''Entf''
  gedrückt werden.}
{Metaknoten wird gelöscht. %IT3 Workflow
}
{nicht spezifiziert.}
{PLE}

\subsubsection{Abhängigkeitskante erstellen (graphisch)}
\usecase{Produktlinienarchitektur geöffnet}
{Über die Palette wird das Werkzeug ''Include Edge'' ausgewählt. Durch
  einen Klick auf ein Blockelement wird der Startpunkt der Kante
  ausgewählt. Der zweite Klick definiert das Zielelement.}
{Die Kante wird eingefügt. Wenn die Kante eine Abhängigkeit von
  Elementen auf unterschiedlichen Hierarchieebenen definiert, wird
  zusätzlich eine synthetische Kante angezeigt.
% IT3 Workflow senden - neue Anhängigkeit
}
{Es wird keine Kante hinzugefügt.}
{PLE}

\subsubsection{Unverträglichkeitskante erstellen (graphisch)}
\usecase{Produktlinienarchitektur geöffnet}
{Über die Palette wird das Werkzeug ''Exclude Edge'' ausgewählt. Durch
  einen Klick auf ein Blockelement wird der Startpunkt der Kante
  ausgewählt. Der zweite Klick definiert das Zielelement.}
{Die Kante wird eingefügt. Wenn die Kante eine Unverträglichkeit von
  Elementen auf unterschiedlichen Hierarchieebenen definiert, wird
  zusätzlich eine synthetische Kante angezeigt.
% IT3 Workflow senden - neue Anhängigkeit
}
{Es wird keine Kante hinzugefügt.}
{PLE}

\subsubsection{Kante löschen}
\usecase{Produktlinienarchitektur geöffnet, Kante ausgewählt}
{Über das Kontextmenu wird die Aktion ''Löschen''
  ausgewählt. Alternativ kann auch die Taste ''Del'' bzw. ''Entf''
  gedrückt werden.}
{Die Kante wird entfernt. %IT3 - Workflow senden
}
{Die Kante wird nicht entfernt.}
{PLE}

\subsubsection{Komponente als \emph{deprecated} markieren}
\usecase{Produktlinienarchitektur geöffnet, Komponente ausgewählt}
{Der Benutzer doppeltklickt auf die Komponente. Ein Dialog öffnet sich,
in dem er die Komponente als ''deprecated'' markieren kann.}
{Die Komponente wird als \emph{deprecated} markiert und mit einem
  Kreuz gekennzeichnet. %IT3 Workflow
}
{nicht spezifiziert.}
{PLE}

\subsubsection{Blockelemente verschieben}
\usecase{Produktlinienarchitektur geöffnet}
{Per Drag\&Drop können alle Blockelemente verschoben werden.}
{Blockelement wird verschoben.}
{nicht spezifiziert.}

\usecase{Produktlinienarchitektur geöffnet, Blockelement ausgewählt}
{Über die Ankerpunkte in der Blockmarkierung kann mittels
  Klicken\&Ziehen die Größe des Blockelements geändert werden.}
{Blockelement wird vergrößert/verkleinert.}
{nicht spezifiziert.}
